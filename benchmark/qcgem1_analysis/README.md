## 1. 测试概述

### 1.1 测试目标

评估 **FlagGems QC-GEM W8A16 / W4A16 Triton Kernel** 相对于 **PyTorch FP16 GEMM** 的性能表现。

### 1.2 量化方案

| 模式 | 权重量化 | 激活值 | Group Size | 量化方式 |
|------|---------|--------|------------|---------|
| **W8A16** | INT8 | FP16 | 128 | per-group min-max 均匀量化，含零偏 |
| **W4A16** | INT4（每字节 2 值） | FP16 | 128 | per-group min-max 均匀量化，含零偏 |

- W4A16: 每 2 个 int4 nibble pack 进 1 byte；Triton kernel 内 on-the-fly unpack + 反量化
- W8A16: 每 byte 1 个 uint8；Triton kernel 内直接反量化
- FP16 参考: 纯 PyTorch GEMM

### 1.3 测试环境

| 项目 | 配置 |
|------|------|
| GPU | NVIDIA H20 (sm_90a) |
| 精度 | FP16 |
| 框架 | PyTorch + FlagGems Triton Kernel |
| Triton 版本 | 见 GemLite 环境 |
| Benchmark 循环 | 10 次计时，3 次 warmup |
| 随机种子 | 42 |

### 1.4 测试 Shape 分类

| 分类 | Shape (M,N,K) | 说明 |
|------|---------------|------|
| Down projection | (256~32768, 3584, 1024) | M 变化，N=3584, K=1024 |
| Up projection | (2048~32768, 1024, 3584) | M 变化，N=1024, K=3584 |
| Gate projection | (2048~32768, 1024, 7168) | M 变化，N=1024, K=7168 |
| Router | (4096~32768, 1024, 128) | M 变化，N=1024, K=128 |

## 3. 性能对比总结

### 3.1 W8A16 vs W4A16 对比分析

| 指标 | W8A16 | W4A16 |
|------|------:|------:|
| 平均加速比 | 0.941× | 0.879× |
| 最大加速比 | 1.020× | 1.017× |
| 最小加速比 | 0.615× | 0.570× |
| ≥1.0 配置数 | 14/21 | 12/21 |
| 平均 TFLOPS | 56.23 | 58.47 |
| 最大 TFLOPS | 93.79 | 94.63 |

### 3.2 分析结论

1. **大 Shape 场景**: 当 M >= 2048 时，QC-GEM 在大多数配置下达到或接近 PyTorch FP16 性能
2. **小 Shape 场景**: 当 M < 2048 时，量化开销相对较大，导致性能下降
3. **Router 场景 (K=128)**: 由于 K 值较小，量化 Kernel 性能明显低于 FP16
4. **W8A16 vs W4A16**: W8A16 在大多数配置下表现优于 W4A16，主要因为 W4A16 需要额外的 INT4 unpack 开销
